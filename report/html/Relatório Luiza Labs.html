Pontos de atenção dos algoritmos de clusterização¶

Para lidar com essa quest√£o, algumas alternativas s√£o sugeridas na literatura cient√≠fica:

  1. 1-of-K Coding [8]: consiste em transformar todas as vari√°veis categ√≥ricas em dummy vars.   2. K-prototypes [8][9]: esse algoritmo √© baseado no KMeans e no KModes. √â adaptado para lidar com um mix de vari√°veis categ√≥ricas e cont√≠nuas, usando medidas de dist√¢ncia adequadas para cada um desses tipos. Os trade-offs desse m√©todo s√£o:

    2.1. Uma medida de dist√¢ncia bin√°ria √© usada para as vari√°veis categ√≥ricas. Essa estrat√©gia n√£o permite capturar a real diferen√ßa entre os valores categ√≥ricos. Ex: a dist√¢ncia entre valores "alto" e "baixo" pode ser "maior" do que a dist√¢ncia entre os valores "alto" e "m√©dio" [1]

    2.2. Apenas um atributo categ√≥rico √© escolhido para pertencer ao centro do cluster, mesmo que os demais elementos categ√≥ricos tenham um papel importante durante o processo de clusteriza√ß√£o [1]

  1. Two-step Method for Clustering Mixed numeric and categorical data (TMCM) [1]: o algoritmo foi desenvolvido para conseguir lidar com vari√°veis categ√≥ricas e num√©ricas simultaneamente na clusteriza√ß√£o.   2. Previsão de estoque¶

O desafio de predizer as vendas dos meses de junho, julho e agosto de 2017 pode ser compreendido como Timeseries Forecasting. Para resolver esse tipo de problema, √© poss√≠vel utilizar modelos estat√≠sticos, econom√©tricos ou modelos de machine learning [7]. Para lidar com esse desafio, precisamos quebrar nossa abordagem em duas partes principais:

  1. Produtos pontuais: esses produtos s√£o bem dif√≠ceis de prever, dado seu hist√≥rico pequeno de compras. √â dif√≠cil identificar os elementos de tend√™ncia e sazonalidade presentes nas s√©ries temporais nesses casos. Dessa forma, usaremos a t√©cnica de exponential smoothing
  2. Demais produtos: nesses casos temos uma amostra maior de hist√≥rico de compras e ser√° mais f√°cil usar modelos que lidam com sazonalidades e tend√™ncias. Aqui, ser√£o comparados dois modelos, exponential smoothing e ARIMA para averiguar qual deles tem uma performance melhor na predi√ß√£o.

Todos os modelos ser√£o avaliados e comparados usando a m√©trica RMSE.

def fill_series(ts, start, end):
    start, end = [datetime.strptime(_, "%Y-%m-%d") for _ in [start, end]]
    total_months = lambda dt: dt.month + 12 * dt.year
    mlist = []
    for tot_m in xrange(total_months(start)-1, total_months(end)):
        y, m = divmod(tot_m, 12)
        mlist.append(datetime(y, m+1, 1).strftime("%Y-%m-01"))
    
    diff = list(set(mlist) - set(ts.index))
    for i in diff: 
        ts = ts.append(pd.Series(0, index=[i]))
        ts = ts.sort_index()
    return(ts)

def create_timeseries(df, product_id, index_col='ym'):
    df2 = df.copy()
    df2['ym'] = df2['capture_date'].apply(lambda x: x.strftime('%Y-%m-01'))
    g = df2.groupby(['ym', 'code']).agg({'quantity': 'sum'}).reset_index()
    aux = g[g['code'] == product_id]
    ts = aux['quantity'].copy()
    ts.index = aux[index_col]
    start = aux[index_col].min()
    end = aux[index_col].max()
    ts = fill_series(ts, start, end)
    return(ts)

def mean_absolute_percentage_error(y_true, y_pred): 
    y_true, y_pred = np.array(y_true), np.array(y_pred)
    return np.mean(np.abs((y_true - y_pred) / y_true)) * 100

def apply_exp_smoothing(ts):
    y_true = ts[-1]
    ts_train = ts[:-1]
    obj = {}
    try:
        fit = SimpleExpSmoothing(ts_train).fit(smoothing_level=0.6,optimized=False)
        y_hat = fit.forecast()
        obj['y_true'] = y_true
        obj['y_hat'] = y_hat
        obj['mape'] = mean_absolute_percentage_error(y_true, y_hat)
        obj['rmse'] = math.sqrt(mean_squared_error([y_true], [y_hat]))
    except:
        obj['y_true'] = np.nan
        obj['y_hat'] = np.nan
        obj['mape'] = np.nan
        obj['rmse'] = np.nan
    return(obj)

# ARIMA
def evaluate_arima_model(X, arima_order):
    size = -1
    train, test = X[:size], [X[size]]
    history = [x for x in train]
    predictions = list()
    for t in range(len(test)):
        model = ARIMA(history, order=arima_order)
        model_fit = model.fit(disp=0)
        output = model_fit.forecast()
        yhat = output[0]
        predictions.append(yhat)
        obs = test[t]
        history.append(obs)
#         print('predicted=%f, expected=%f' % (yhat, obs))
    mse = mean_squared_error(test, predictions)
    rmse = math.sqrt(mse)
    return(rmse)

# evaluate combinations of p, d and q values for an ARIMA model
def evaluate_models(dataset, p_values, d_values, q_values):
    dataset = dataset.astype('float32')
    best_score, best_cfg = float("inf"), None
    for p in p_values:
        for d in d_values:
            for q in q_values:
                order = (p,d,q)
                try:
                    mse = evaluate_arima_model(dataset, order)
                    if mse < best_score:
                        best_score, best_cfg = mse, order
#                     print('ARIMA%s RMSE=%.3f' % (order,mse))
                except NameError as e:
                    print e
                    continue
                except ValueError as e:
                    print e
                    continue
                except:
                    print "Unexpected error:", sys.exc_info()[0]
                    continue
#     print('Best ARIMA%s RMSE=%.3f' % (best_cfg, best_score))
    return(best_cfg)

def apply_arima(ts):
    obj = {}
    # evaluate parameters
    p_values = range(0, 2)
    d_values = range(0, 2)
    q_values = range(0, 2)
    arima_cfg = evaluate_models(ts, p_values, d_values, q_values)
    
    ts = ts.astype('float32')
